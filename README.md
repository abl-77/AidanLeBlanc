## CrowdSearch (Full-Stack Crowd Forecasting Platform)

**Description:**  
CrowdSearch is a full-stack web application that helps students make better decisions about where and when to go on campus by providing **real-time and predicted crowd levels for popular locations**. The system combines **user-reported data with historical trends** to forecast how crowded locations such as gyms, libraries, and dining areas will be at different times of day.

Users can view crowd conditions across campus and **submit crowd reports when visiting locations**, creating a community-driven dataset that continuously improves prediction accuracy.

**Key Features**

- **Interactive campus map** displaying crowd levels at major locations
- **Crowd reporting system** triggered when a user is near a location of interest
- **Crowd forecasting model** predicting crowd levels based on historical reports grouped by weekday and hour
- **Community-driven data collection** that improves predictions over time
- Helps users decide:
  - when to visit a location
  - whether it will be crowded
  - if an alternative location may be better

**System Architecture**

- **Frontend:** React interface with interactive map visualization  
- **Backend:** Flask API handling data collection and forecasting logic  
- **Data Pipeline:** Aggregates user reports and computes predicted crowd levels using historical reporting statistics

**Technologies**

React, JavaScript, Flask, Python, Node.js, REST APIs

## Learned Subgoal Navigation for Autonomous Agents (Machine Learning)

**Description:**  
This research project explores how **learned models can guide navigation in partially observable environments**. Inspired by research on learned subgoal planning, the project builds a simulation framework to compare classical path planning algorithms with machine-learning-guided navigation.

The system evaluates how learned models perform in different maze environments and introduces a **similarity-based planning method** that dynamically switches between learned models and classical planners depending on environment familiarity.

**Key Features**

- Procedural **maze generation using Kruskal's algorithm**
- Baseline **Dijkstra-based optimistic planner**
- Fully connected neural networks trained to predict **subgoal success probabilities**
- Dataset generation pipeline producing **50,000+ training samples**
- **Environment similarity classifier** that adapts planning strategy in unfamiliar environments
- Performance evaluation across **guided, anti-guided, and hybrid maze environments**

**Technologies**

Python, PyTorch, Machine Learning, Path Planning Algorithms, Data Generation
